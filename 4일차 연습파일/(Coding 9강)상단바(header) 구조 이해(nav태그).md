
웹사이트 상단바(헤더)를 만들 때, 코드를 어떻게 구성해야 할까? 상단바는 크게 **세 덩어리(**메뉴**, **로고**, **사용자 메뉴**)**로 나누고, 메뉴는 <nav> 태그, 로고는 <h1> 태그, 사용자 메뉴는 <div> 태그를 쓰는 게 일반적인 약속이야.
## 1. 파일 관리, 왜 중요할까? 

- 코딩할 때 파일 관리를 잘하는 건 마치 책상 정리랑 똑같아. 
  - 책상이 깔끔하면 필요한 물건을 바로 찾을 수 있잖아? 
  - 코딩도 마찬가지로, 파일이 잘 정리되어 있으면 나중에 코드를 고치거나 찾을 때 훨씬 편해. 
- 특히 AI (인공지능)랑 같이 작업할 때는 더 중요해. 
  - AI한테 코드를 보여줄 때, 파일이 너무 많거나 복잡하면 AI도 힘들어하고 시간도 오래 걸려. 
  - 그래서 처음부터 파일 관리를 잘 해두면 AI를 더 똑똑하게 활용할 수 있어. 

## 2. 웹사이트 상단바(헤더)는 어떻게 만들까? 

- 웹사이트 맨 위에 있는 부분을 **상단바** (헤더)라고 부르는데, 여기에는 보통 세 가지 중요한 부분이 들어가. 
  - 메뉴: 다른 페이지로 이동할 수 있는 버튼들이 모여있는 곳이야. 
  - 로고: 웹사이트의 이름이나 브랜드를 나타내는 그림이나 글자야. 
  - 사용자 메뉴: 로그인, 회원가입, 내 정보 같은 기능들이 있는 곳이야. 
- 이 세 가지를 각각 따로따로 만들어서 합치는 게 일반적인 방법이야.

## 3. 상단바의 세 가지 부분, 어떤 태그를 쓸까? 

- 웹사이트를 만들 때, 각 부분에 맞는 태그 (HTML에서 쓰는 약속된 이름표)를 사용하는 게 중요해. 
  - 마치 물건마다 이름표를 붙여서 어떤 물건인지 알 수 있게 하는 것과 같아.
- 메뉴 부분은 <nav> (내비게이션) 태그를 사용해. 
  - <nav>는 "여기에는 다른 페이지로 이동하는 메뉴들이 있어요!"라고 알려주는 태그야. 
- 로고 부분은 <h1> (헤딩 1) 태그를 사용해. 
  - <h1>은 "이 페이지에서 가장 중요한 제목이에요!"라는 뜻인데, 웹사이트에서는 보통 로고에 이 태그를 써서 중요하다고 알려주는 약속이 있어. 
- 사용자 메뉴 부분은 <div> (디비전) 태그를 사용해. 
  - <div>는 "여기는 그냥 어떤 내용을 묶어 놓은 공간이에요!"라는 뜻이야. 특별한 의미는 없지만, 여러 요소를 묶을 때 많이 써. 
  - 만약 사용자 메뉴가 여러 개라면, <div>에 클래스 이름 (class name)을 붙여서 "이 <div>는 사용자 메뉴야!"라고 표시해 줄 수 있어. 
  - 예를 들어, user-box 같은 이름을 붙여서 나중에 찾기 쉽게 만드는 거지.
