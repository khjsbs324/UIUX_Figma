코딩 공부할 때 가장 중요한 건 뭘까? **HTML만 외워서 짜거나 CSS만 외워서 짜지 말고, HTML과 CSS를 번갈아 가면서 작성**하는 거야. 그래야 나중에 더 어려운 자바스크립트 같은 것도 잘 할 수 있어.
## 1. VS Code와 Figma 다시 연결하기 
<gen-svg-visual description="A flowchart showing the steps to log out and log back into Figma within VS Code. It starts with 'Click 3 dots (logout)', then 'Click Logout'. Next, 'Click Login (blue button)', then 'Click Open (black button)'. Finally, 'VS Code opens Figma'. Arrows connect each step sequentially." />

- **피그마(Figma)**가 **VS **코드**(Visual Studio Code)**에서 잘 안 보일 때 다시 연결하는 방법이야. 
  - 먼저, 피그마에서 **로그아웃**을 해야 해. 
  - 피그마 화면에서 점 세 개를 누르고 '로그아웃'을 누르면 돼. 
  - 그다음, 다시 **로그인**을 해줘. 
  - 파란색 '로그인' 버튼을 누르면 돼. 
  - 그리고 '오픈(Open)' 버튼을 누르면 돼. 
  - 까만색 버튼이 보일 거야. 
  - 마지막으로, **VS 코드**에서 피그마를 열어주면 돼. 
  - VS 코드에 '오픈' 버튼이 보이면 눌러주면 돼. 
  - 그러면 왼쪽에 최근에 썼던 파일들이 보이고, 우리가 보려는 피그마 파일이 열릴 거야. 

## 2. 피그마 화면에서 크기 확인하는 방법 
<gen-svg-visual description="A diagram showing two ways to check the size (width and height) of elements in Figma. This illustrates direct clicking vs. using the layer panel." />

- 피그마에서 디자인 요소들의 **크기(**너비와 높이**)**를 확인하는 두 가지 방법이 있어. 
  - **직접 클릭해서 확인하기**: 
  - 피그마 화면에서 원하는 부분을 **클릭**하면, 바로 아래쪽에 그 부분의 너비와 높이가 나타나. 
  - 이 방법이 가장 빠르고 편해서 많이 사용해. 
  - 레이어** 아이콘으로 확인하기**: 
  - 화면 위쪽에 있는 **레이어 아이콘**을 누르면, 디자인 요소들이 층층이 쌓여 있는 목록이 보여. 
  - 이 목록에서 하나씩 눌러보면 각각의 크기를 확인할 수 있어. 

## 3. 코딩 작업 환경 설정하기 
<gen-svg-visual description="A layout diagram showing an optimal coding environment setup. The screen is divided into three main sections. The left section is labeled 'VS Code (HTML & CSS)' with a small icon representing code. The middle section is labeled 'Figma Screen' with a Figma logo. The right section is labeled 'Live Server (Preview)' with a browser icon. Arrows indicate that changes in VS Code are reflected in Live Server, and Figma provides design references." />

- 코딩할 때 효율적으로 작업하기 위한 화면 배치 방법이야. 
  - 화면을 크게 세 부분으로 나누면 좋아. 
  - **왼쪽**: HTML 코드와 CSS 코드를 작성하는 **VS **코드 화면을 둬. 
  - **오른쪽**: 디자인을 참고할 피그마 화면을 둬. 
  - **아래쪽 또는 다른 곳**: 코드를 바꾸면 바로 결과가 보이는 **미리 보기(**라이브 서버**)** 화면을 띄워놔. 
  - 만약 미리 보기 화면이 잘 안 뜨면, 배경색**(background-color)**을 회색(gray)으로 바꿔서 화면이 제대로 나오는지 먼저 확인해야 해. 

## 4. 코딩 공부할 때 꼭 지켜야 할 규칙 


- 코딩을 잘 배우려면 몇 가지 중요한 규칙을 지켜야 해. 
  - **HTML만 쭉 쓰지 마**: 
  - HTML 코드만 외워서 쭉 작성하면, 나중에 자바스크립트**(JavaScript)** 같은 더 어려운 코드를 짜기 힘들어져. 
  - HTML과 CSS는 어떻게든 할 수 있지만, 자바스크립트부터는 막히게 될 거야. 
  - **HTML과 CSS를 번갈아 가면서 작성해**: 
  - HTML 코드를 조금 쓰고, 바로 CSS로 꾸며보고, 다시 HTML을 추가하는 식으로 왔다 갔다 하면서 작업해야 해. 
  - 이렇게 해야 코드가 어떻게 작동하는지 더 잘 이해할 수 있어. 
  - 주석**(comment)을 적극적으로 사용해**: 
  - 코드에 설명을 달아두는 **주석**을 많이 사용하면, 나중에 코드를 다시 볼 때나 다른 사람이 볼 때 이해하기 쉬워. 
  - **배경색과 글자색은 꼭 넣어**: 
  - 코드가 제대로 적용되고 있는지 확인하려면, **배경색(background-color)**과 **글자색(color)**을 필수로 지정해야 해. 
  - 이게 없으면 코드가 잘 작동하는지 알기 어려워. 
  - **모르는 건 바로 질문해**: 
  - 코딩 공부는 모르는 부분을 혼자 끙끙 앓기보다, 질문해서 해결하는 게 중요해. 
  - 아주 사소한 것이라도 괜찮으니, 헷갈리거나 모르는 게 있으면 바로 손들고 물어봐야 해. 

## 5. 코드 보는 것을 두려워하지 마 
<gen-svg-visual description="A simple diagram showing a magnifying glass over a piece of code, with an arrow pointing to a smiling face. Text says 'Looking at code is good!' and 'It helps your eyes get used to it.' This visual encourages students to review code without fear." />

- 다른 사람의 코드를 보거나, 이전에 작성했던 코드를 다시 보는 것을 두려워하지 마. 
  - 코드를 계속 보면 눈에 익어서 나중에 직접 코드를 짤 때 훨씬 도움이 될 거야. 
  - 오히려 코드를 많이 보는 것이 실력 향상에 좋아.
