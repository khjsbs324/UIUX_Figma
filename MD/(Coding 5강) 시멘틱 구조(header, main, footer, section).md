
웹사이트를 만들 때 시멘틱 구조**(Semantic Structure)**가 왜 중요할까? 웹사이트를 머리(header), 몸통(main), 발(footer)처럼 의미 있는 구역으로 나누면, 검색 엔진이 정보를 더 잘 이해하고, 다른 개발자들도 코드를 쉽게 파악할 수 있어. 마치 집을 지을 때 시멘트로 튼튼하게 만드는 것처럼, 웹사이트의 **기본 뼈대를 탄탄하게** 만드는 거라고 생각하면 돼.
## 1. 웹사이트의 기본 구조: 머리, 몸통, 발

Each section is labeled with its corresponding HTML tag (e.g., <header>, <main>, <footer>). Arrows indicate the top-to-bottom flow of these sections." />

- 웹사이트는 보통 세 부분으로 나눌 수 있어. 마치 사람 몸처럼 **머리, 몸통, 발**로 구성되어 있다고 생각하면 돼. 
  - **머리 (**Header**)**: 웹사이트의 가장 위에 있는 부분이야. 보통 로고나 메뉴 같은 중요한 정보가 들어가. 
  - **몸통 (**Main**)**: 웹사이트의 핵심 내용이 들어가는 부분이야. 우리가 보고 싶어 하는 정보들이 여기에 담겨. 
  - **발 (**Footer**)**: 웹사이트의 가장 아래에 있는 부분이야. 회사 정보나 저작권 표시 같은 내용이 주로 있어. 

## 2. 웹사이트의 뼈대, 시멘틱 구조 (Semantic Structure) 


- 웹사이트를 만들 때 이렇게 의미 있는 부분으로 나누는 걸 **시멘틱 구조**라고 불러. 
  - 시멘틱 구조는 마치 집을 지을 때 **시멘트로 튼튼하게 뼈대를 만드는 것**과 같아. 
  - 이렇게 뼈대를 잘 만들어 놓으면, 웹사이트가 더 튼튼하고 알아보기 쉬워지는 거야. 

## 3. 시멘틱 태그로 웹사이트 구역 나누기

- 웹사이트의 각 부분은 특별한 이름표(태그)를 가지고 있어. 
  - <header> 태그: 웹사이트의 머리 부분을 나타내. 항상 웹사이트의 가장 위에 있어야 해. 
  - 머리가 발밑에 있으면 이상하잖아? 웹사이트도 마찬가지야. 
  - <main> 태그: 웹사이트의 몸통, 즉 주요 내용이 들어가는 부분이야. 
  - 이 안에 <section> 태그를 사용해서 내용을 여러 구역으로 더 나눌 수 있어. 
  - <footer> 태그: 웹사이트의 발 부분을 나타내. 항상 웹사이트의 가장 아래에 있어야 해. 
  - 이 태그들을 사용해서 웹사이트를 만들면, 검색 엔진이 정보를 더 잘 이해하고, 다른 개발자들도 코드를 쉽게 파악할 수 있어. 

## 4. 코드 정리의 중요성 


- 코드를 작성할 때는 깔끔하게 정리하는 게 정말 중요해. 
  - 불필요한 빈 공간이나 잘못된 들여쓰기는 코드를 읽기 어렵게 만들고, 용량을 낭비할 수 있어. 
  - 코드를 정리하는 단축키를 활용하면 훨씬 쉽게 깔끔한 코드를 만들 수 있어. 
  - 예를 들어, Shift + Home 키를 누르면 한 줄을 선택할 수 있고, Alt + 윗방향키를 누르면 선택한 줄을 위로 옮길 수 있어. 
  - 이렇게 정리된 코드는 다른 사람이 보기도 좋고, 나중에 수정하기도 훨씬 편해.
