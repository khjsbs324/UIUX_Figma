
깃허브 데스크탑 앱으로 Git Commit**(변경사항 기록)**과 Git Push**(원격 저장소에 올리기)**를 어떻게 하는 걸까? 변경된 파일을 선택하고 커밋 메시지를 작성한 뒤, 'Push origin' 버튼을 누르면 내 작업 내용이 깃허브에 올라가고, 깃허브 페이지에서 최종 결과물을 바로 확인할 수 있어.
## 1. 내 작업물 올릴 준비하기 (Git Commit) 
<gen-svg-visual description="A diagram showing the process of committing changes. It starts with 'Changed Files' (files that have been modified). An arrow points to 'Staging Area' (files selected to be committed), and then another arrow points to 'Local Repository' (where committed changes are stored on your computer). This illustrates how changes move from being just modified to being ready to be saved, and then finally saved locally." />

- 깃허브 데스크톱** 앱**에서 내가 바꾼 파일들을 **선택**하는 과정이야. 
  - 마치 내가 오늘 그린 그림들 중에서 "이 그림들을 앨범에 넣어야지!" 하고 고르는 것과 같아.
  - '체인지(Changes)' 목록에서 내가 올리고 싶은 파일들을 고르고, '+' 버튼을 눌러주면 돼. 
  - 이렇게 선택된 파일들은 이제 **커밋(Commit)**할 준비가 된 거야. 

## 2. 내 작업물에 이름표 붙이기 (Commit Message) 
<gen-svg-visual description="A visual representation of a commit message. It shows a 'Title' field at the top, followed by a larger 'Description' field below it. This illustrates the two main parts of a commit message: a short, clear title and a more detailed explanation." />

- 내가 고른 파일들이 어떤 내용인지 **간단하게 설명**하는 단계야. 
  - 앨범에 사진을 넣을 때 "2024년 여름휴가"라고 제목을 붙이고, "바닷가에서 신나게 놀았음"이라고 설명을 쓰는 것과 비슷해.
  - 제목과 **본문**을 써서 다른 사람들이나 미래의 내가 이 작업이 뭘 바꾼 건지 쉽게 알 수 있게 하는 거지. 
  - 예를 들어, "독타입 선언하고 CSS 파일 만들기"처럼 제목을 쓰고, "오늘은 독 타입을 선언하고 CSS 파일 만들어서 연결해 봄"처럼 자세히 쓰는 거야. 

## 3. 내 작업물을 깃허브에 올리기 (Git Push) 


- 이제 내가 준비하고 이름표까지 붙인 작업물들을 **깃허브 웹사이트**에 실제로 올리는 단계야. 
  - 마치 내가 만든 앨범을 친구들과 공유하는 온라인 갤러리에 업로드하는 것과 같아.
  - '커밋(Commit)' 버튼을 누른 다음, '푸시 오리진(Push origin)'이라는 파란색 버튼을 누르면 돼. 
  - 이 버튼을 누르면 내 컴퓨터에 있던 작업 내용이 깃허브 웹사이트로 전송되는 거야. 
  - 작업이 성공적으로 올라가면, 깃허브 데스크톱 앱에서는 파일 목록 대신 **커밋된 내용**이 보일 거야. 

## 4. 내 작업물이 잘 올라갔는지 확인하기 


- 내 작업물이 깃허브 웹사이트에 제대로 올라갔는지 확인하는 방법이야. 
  - 온라인 갤러리에 사진을 올렸으면, 실제로 갤러리에 들어가서 사진이 잘 보이는지 확인하는 것과 같지.
- **깃허브 웹사이트**로 이동해서 확인해봐. 
  - 처음에는 바로 바뀌지 않을 수도 있어.
  - '액션(Actions)' 탭을 눌러보면, 동그라미가 빙글빙글 돌아가는 걸 볼 수 있을 거야. 
  - 이 동그라미가 초록색 체크** 표시**로 바뀌면, 내 작업물이 깃허브에 완벽하게 올라간 거야! 
- 내 웹사이트가 어떻게 바뀌었는지 보려면: 
  - '세팅(Settings)' 탭으로 가서 '페이지(Pages)'를 누르고, '뷰 사이트(View Site)' 버튼을 누르면 돼. 
  - 그럼 내가 올린 최신 작업 내용이 반영된 웹사이트를 직접 볼 수 있을 거야.
