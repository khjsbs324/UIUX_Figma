
로컬 저장소(내 컴퓨터에 있는 폴더)를 깃허브 데스크탑에 연결할 때 Add local repository와 Clone repository의 차이점은 뭘까? 이미 내 컴퓨터에 있는 폴더를 연결할 때는 'Add local repository'를, 깃허브에 있는 프로젝트를 새로 가져올 때는 'Clone repository'를 쓰는 거야.
## 1. 깃허브 데스크탑에 폴더 연결하기: 두 가지 방법 

- 깃허브 데스크탑에 내 컴퓨터에 있는 폴더(프로젝트)를 연결하는 방법은 크게 두 가지가 있어.
  - Add local repository** (**애드** **로컬** **리포지토리**)**: 이미 내 컴퓨터에 있는 폴더를 깃허브 데스크탑에 추가하는 방법이야. 
  - 이건 마치 네 방에 이미 있는 장난감 상자를 "이것도 내 장난감 상자야!" 하고 목록에 추가하는 거랑 비슷해.
  - Clone repository** (클론 리포지토리)**: 깃허브 웹사이트에 있는 프로젝트를 내 컴퓨터로 새로 가져오는 방법이야. 
  - 이건 마치 친구 집에 있는 멋진 장난감을 "나도 이거 똑같이 하나 만들래!" 하고 새로 만드는 거랑 비슷해.

## 2. 언제 'Add local repository'를 쓸까? 

- 이미 내 컴퓨터에 프로젝트 폴더가 있을 때 쓰는 방법이야. 
  - 예를 들어, 네가 이미 컴퓨터에서 어떤 그림을 그리고 있었는데, 이제 그 그림을 깃허브 데스크탑으로 관리하고 싶을 때 쓰는 거지.
  - 이 방법은 깃허브에 있는 내용을 가져오는 게 아니라, 내 컴퓨터에 있는 내용을 깃허브 데스크탑에 "이 폴더를 관리해 줘!" 하고 알려주는 거야. 
- 이 방법을 쓰면 깃허브에 있는 내용이 내 컴퓨터에 덮어씌워지는 게 아니라, 내 컴퓨터에 있는 내용이 깃허브 데스크탑에 연결되는 거야. 

## 3. 언제 'Clone repository'를 쓸까? 

- 깃허브 웹사이트에 있는 프로젝트를 내 컴퓨터로 새로 가져오고 싶을 때 쓰는 방법이야. 
  - 예를 들어, 친구가 깃허브에 올려놓은 멋진 게임 코드를 "나도 이 게임 해보고 싶다!" 하고 내 컴퓨터로 가져올 때 쓰는 거지.
  - 이 방법은 내 컴퓨터에 그 프로젝트 폴더가 아예 없을 때, 깃허브에서 처음으로 가져와서 새 폴더를 만들 때 사용해. 



## 4. 'Add local repository' 사용 방법 

- 깃허브 데스크탑에서 'Add local repository'를 누른 다음, 내 컴퓨터에 있는 프로젝트 폴더의 위치(경로)를 알려주면 돼. 
  - 마치 "내 장난감 상자는 저기 저 방에 있어!" 하고 알려주는 거랑 같아.
- 이렇게 하면 깃허브 데스크탑이 그 폴더를 인식하고 관리하기 시작하는 거야. 
  - 중요한 건, 이 방법은 깃허브에 있는 코드를 가져와서 내 컴퓨터에 덮어씌우는 게 아니라는 점이야. 

## 5. 두 방법의 핵심 차이점 

- Clone** (클론)**: 깃허브에 있는 걸 **새로 가져와서** 내 컴퓨터에 만드는 거야. (초기화, 처음 시작할 때) 
  - "아무것도 없으니까 깃허브에서 새롭게 시작하자!" 할 때 쓰는 거지.
- **Add local (애드 로컬)**: 내 컴퓨터에 **이미 있는 걸** 깃허브 데스크탑에 연결해서 관리하는 거야. (이미 작업 중일 때) 
  - "이미 작업하고 있던 게 있는데, 이제 깃허브 데스크탑으로 관리하고 싶어!" 할 때 쓰는 거지.
- 그래서 이 두 가지는 완전히 다른 상황에서 사용되는 거야.
